# PRメッセージ詳細ガイド

## テンプレート

```markdown
## 概要
[変更内容を1-3行で説明]

## レビュー観点
- [確認してほしいポイント1]
- [確認してほしいポイント2]

## 関連Issue
- Closes #123
- Refs #456
```

## 各セクションの書き方

### 概要

変更の目的と内容を簡潔に説明する。

**良い例：**
```markdown
## 概要
ユーザー認証にOAuth2.0を導入し、Googleアカウントでのログインを可能にしました。
既存のメールアドレス認証と併用できます。
```

**悪い例：**
```markdown
## 概要
認証を変更しました。
```

ポイント：
- 何を変更したかだけでなく、なぜ変更したかも記載
- 技術的な詳細より、ユーザー/ビジネス視点での価値を優先
- 影響範囲が分かるように記述

### レビュー観点

レビュアーに特に確認してほしいポイントを箇条書きで記載。

**例：**
```markdown
## レビュー観点
- セキュリティ: OAuthトークンの保存方法は適切か
- パフォーマンス: 認証フローのレスポンス時間
- エラーハンドリング: OAuth認証失敗時のフォールバック処理
- 既存機能への影響: メールアドレス認証との共存
```

観点の例：
- **セキュリティ**: 脆弱性、権限チェック、入力バリデーション
- **パフォーマンス**: クエリ最適化、レスポンス時間
- **設計**: アーキテクチャ、責務分離、拡張性
- **エラーハンドリング**: エッジケース、例外処理
- **テスト**: カバレッジ、テストケースの妥当性
- **互換性**: 既存機能への影響、後方互換性

### 関連Issue

関連するIssueやチケットへのリンクを記載。

```markdown
## 関連Issue
- Closes #123    ← マージ時にIssueを自動クローズ
- Fixes #124     ← マージ時にIssueを自動クローズ
- Refs #456      ← 参照のみ（クローズしない）
- Related to #789 ← 参照のみ（クローズしない）
```

## PRタイトルの書き方

コミットメッセージと同様にConventional Commits形式を推奨：

```
feat(auth): OAuth2.0によるGoogleログインを追加
fix(cart): カート合計金額の計算誤りを修正
docs(api): RESTful APIのドキュメントを追加
```

## PRサイズのガイドライン

| サイズ | 変更行数 | 推奨 |
|--------|----------|------|
| XS | ~10行 | ✓ |
| S | ~100行 | ✓ |
| M | ~300行 | △ 可能なら分割 |
| L | ~500行 | × 分割を検討 |
| XL | 500行~ | × 必ず分割 |

大きなPRは：
- レビューに時間がかかる
- バグの見落としリスクが高い
- コンフリクトが発生しやすい

## サンプルPR

```markdown
## 概要
ユーザーダッシュボードに月次レポート機能を追加しました。

- 過去12ヶ月のデータをグラフで表示
- CSV形式でのエクスポート機能
- 日付範囲のフィルタリング

## レビュー観点
- グラフ描画のパフォーマンス（1000件以上のデータ）
- CSVエクスポート時のメモリ使用量
- 日付フィルターのバリデーション

## 関連Issue
- Closes #234
- Refs #198
```
